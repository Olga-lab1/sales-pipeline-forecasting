An executive-grade revenue analytics dashboard that answers the four questions every CRO asks in a forecast call:

1. **Will we hit the number this quarter?** — Holt-Winters revenue forecast with 80% confidence intervals and a quarter-end projection gauge vs. quota
2. **Is the pipeline healthy enough for next quarter?** — weighted pipeline coverage ratios and close-date slippage tracking
3. **Where does the funnel leak?** — stage-by-stage conversion built from the furthest stage each deal reached
4. **Why do we lose, and who wins?** — dollar-quantified loss reasons and a rep leaderboard

## Highlights

- **Honest forecasting.** The model is validated with rolling-origin backtesting, and the dashboard displays its out-of-sample MAPE next to the chart — a forecast without a stated error rate is just a drawing.
- **Realistic synthetic data.** A fully reproducible generator (`src/generate_data.py`) creates 6,000 Salesforce-style opportunities with seasonality (Q4 push, summer dip), segment economics (Enterprise ACV ≫ SMB), rep skill variation, lead-source lift, close-date slippage, and stage-of-loss tracking.
- **CRM-standard metrics.** Stage-weighted pipeline, coverage ratios vs. quota, win rates, cycle times, and loss-reason revenue — the vocabulary of a real RevOps team.
- **Engineering hygiene.** Typed, documented modules; 12 pytest tests; ruff linting; GitHub Actions CI.

## Quick start

```bash
git clone https://github.com/Olga-lab1/sales-pipeline-forecasting.git
cd sales-pipeline-forecasting
pip install -r requirements.txt
python src/generate_data.py        # optional: regenerate the dataset
streamlit run app/dashboard.py
```

Then open http://localhost:8501. Use the sidebar to filter by segment/region, set the quarterly quota, and change the forecast horizon.

## Methodology notes

**Why Holt-Winters?** Monthly bookings in B2B SaaS are dominated by two signals — growth trend and annual seasonality — which additive exponential smoothing captures directly. It is fast, dependency-light, and its components (level/trend/season) are explainable to a non-technical audience, which matters more in a boardroom than a marginal accuracy gain from a black-box model.

**Confidence intervals** are empirical: the 10th/90th percentiles of in-sample residuals, widened by √horizon. This avoids the over-tight intervals that parametric assumptions often produce on short business series.

**Backtesting** uses rolling-origin evaluation: the model is refit at six historical cut points and scored on the following three months. The resulting MAPE is shown in the UI so consumers of the forecast know exactly how much to trust it.

**Funnel construction** uses the *furthest stage reached* per deal (open stage, stage-at-loss for lost deals, or Closed Won), avoiding the classic mistake of computing conversion from a snapshot of current stages.

## Data disclaimer

All data in this repository is **synthetic**, generated by `src/generate_data.py` with a fixed random seed. No real company, customer, or personal data is used.
